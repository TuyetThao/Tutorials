Basic Git Commands




Command Function
git init Creating a new Git repository or creating an existing one. git init is understood as a directory with many subdirectories and sample files in it. When running a new git init on an existing repository, it will not overwrite anything that already exists(GitHub, 2020).
git clone Download a project with all files from the Github to the user computer.
git add Add files into the staging area of ​​Git to track and snapshot of code changes to include them in project history. Each change is referred to as the next snapshot. This staging and committing helps developers manage things without changing the way they write the code (Reshma, 2020).
git pull Update and copy all files from the master branch of remote repository to the local repository.
git commit Record the snapshot to the project’s history to finish the process.
git push Creating a new Git repository or creating an existing one. git init is understood as a directory with many subdirectories and sample files in it. When running a new git init on an existing repository, it will not overwrite anything that already exists(GitHub, 2020).
git config Copyright your code by showing it through author name and email address, so that when you have any action with the joint project, it helps other members know whose part is.
git log This is used as a tool to review or re-read the project's history. In short, git log is a record of commits.
git status Lists the modified but not tracked folders / files to be added to the local repository.
git remote Used to connect the local repository to a remote server so that all members can exchange their changes.
