What Is Branching?


git branch

This is a branching feature in the develop the project process. When developers want to fix errors, or add any features, this change will be encapsulated in a branch, which makes it easier for the developer to control the code and not affect locally. When new features or changes in the branch are tested and run completely, the developer merges the new part into the master source code.

Instead of the direct merge, the developer will create a pull request. The pull request will show the changes to the code for all group members, all the members will review all the code and accept it.


git checkout

When we want to change from one branch to another in the project, we use the checkout command. It's different from the drive git clone command where git clone will make a copy of the code to combine the code into the remote repository, while the git checkout switches between versions (Atlassian, n.d).


git merge

This command is used to convert small branches into a single branch, also known as master.
